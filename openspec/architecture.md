The module is deliberately NOT relocated (react stays byte-sealed) — ADR-02 registers the now-realised rename/placement debt; the eventual peer-module split (ADR-01's arm b) remains deferred, gated on owner ratification. Still a placement/cohesion debt, NOT a dependency-direction breach — both edges are dialect → core (sanctioned; FIT-02 holds, the import is `../../core/*`).

Two DELIBERATE reject shapes pinned distinct (validation path-less vs collision path-ful). Signature FROZEN, `.d.ts` unchanged. Leaf-isolated; FIT-02 forbids a sibling-dialect import; the new `../../core/{jsx-name-validator,reject-tail,dialect-handle}` edges are all sanctioned dialect → core.

Note 1 (stdio-engine-client): the engine edge is HOST-INITIATED — the engine spawns the runner (or dynamic-imports the bridge in its own process) and sends the first frame (`{method:"ready", protocolVersion}`); the SDK then owns the raw read loop (ADR-0054, single reader) and issues reverse callbacks. This REVERSES the old "SDK dials the engine" assumption; the wire is NOT JSON-RPC 2.0 — it is the repo's own framed JSON `{type:"request"|"response", id:"s<n>", method, params}` grammar, normatively pinned in docs/engine-sdk-wire-spec.md (fit-33/fit-34 pin the constants against that doc at test time).

Note 3: `./conformance`'s run-vehicle satisfies the port SHAPE via a LOCAL structural interface — never names `EngineClient` (FIT-10) and is never re-exported (FIT-08). It is a SEPARATE thin implementation from `src/testing/contract-fake.ts`.
Note 4: `./testing` and `pbuilder-codegen` retain their in-memory/write-contained postures; `./testing`'s `defineFactory` edge plus `src/core/**`/`src/conformance/**`/`src/transport/runner.ts` are the ONLY sanctioned production paths to `defineFactory` (FIT-29).

## Pattern
- @pbuilder/sdk: custom — layered modules behind an IR-seam port, dialect leaves, and a transport leaf realizing the port
  Evidence: `src/commons/` (author verbs) → `src/core/` (Session, DirectiveFactory, handle-state, dialect-handle) → `src/core/wire.ts` (Directive/Batch shapes + cap seam) → `src/core/engine-client.ts` (single `EngineClient` port) ← `src/transport/stdio-engine-client.ts` (first real implementer). `src/dialects/{typescript,react}/`, `src/dry-run/`, `src/testing/`, `src/scaffold/`, `src/transport/` sit beside `core` as deliberately-isolated leaves. No `adapters/`/`ports/`/`usecases/`/`domain/` scaffold; the seam is one interface.
- **`src/transport/` cluster (stdio-engine-client, ADRs 0053-0059)**: a transport leaf with strict internal ownership rules — `framing.ts` is the single owner of encode/decode + the only sanctioned stdout reference (fit-30/fit-31); `core/wire.ts` owns the single `> BATCH_CAP_BYTES` comparison both legs share (fit-32); `runner.ts` is the ONE composition root both entry vehicles share; `stdio-engine-client.ts` is the only transport file allowed to name `EngineClient` (fit-10 +1). Dependency direction: transport → core (types + cap measurers + `defineFactory` via the sanctioned-caller idiom), never core → transport; bin → transport → core (FIT-15 one-way holds).
- **`src/dialects/typescript/` cluster (stage-5/5b; `addImport` hardened ts-addimport-collision)**: author-facing dialect leaf; `ast.ts` sole ts-morph importer; `ops.ts` five ops + `assertNoCollision`. `addImport` is now a validation-gated 4-step op (inline `assertValidImportBinding` from `core/jsx-name-validator.ts` first, then idempotency / collision (`isValueNamespaceClaimed`) / directive-aware create) rather than a naive first-match merge; verdict-parity with the react leaf's `addImport` is guarded by FIT-41 (mirror-in-leaf, ADR-01) — react byte-sealed, ZERO `src/dialects/react/**` diff.
- **`src/dialects/react/` cluster (react-dialect, ADR-01..05)**: author-facing dialect leaf, structurally the mirror of the TypeScript leaf; a synchronous `.tsx`-only `find` gate; `ast.ts` the leaf's sole ts-morph importer (fail-loud on syntactic diagnostics); `ops.ts` a closed two-op v1 vocabulary (`setJsxProp`+`addImport`), every op wrapped in `validatedOp` (validate-before-mutate) with a no-echo reject-tail. DUPLICATES the TS leaf's fidelity primitives rather than importing them (ADR-01, leaf-isolation cost).
- **`src/core/dialect-handle.ts` (coalescing seam, ADR-0037 + amendment)**: `#tail` queue, one live AST, one open lazy-getter `modify` directive; `#invokeContained` + WeakSet-brand discrimination; run-scoped first-wins poison flag `RunContext.runFailure` (fail-closed, FIT-21 one-way guard); mutation gate; WeakMap path channel `handlePathFor(ast)`.
- **`src/conformance/` cluster (stage-5/5b)**: real `Promise<void>` bodies driving REAL `defineFactory` runs against the kit-internal run-vehicle; six mandatory adversarial samples + real-base probe, no fixture opt-out; leaf-isolation for third-party dialects stays a DOCUMENTED LIMIT.
- **`src/core/schema/` cluster (stage-4)**: cohesive core module behind the IR seam (array model, hand-rolled JSON locator), shared by runtime + bin + runner.
- **`src/testing/` cluster (stage-4b; bare-factory-migration)**: author-facing test-harness leaf; `contract-fake.ts` is the normative `ContractFake implements EngineClient` (FIT-18 parity-by-identity), cap-parity-locked to the real client via `exceedsEmitBatchBudget`.

## Data flow
- Author verb → `DirectiveFactory.{op}` → `Session.buffer` → (read() or run-end) → `Session.flush()` → `EngineClient.emit(Batch)` → engine.
- Read path: `read()` → `Session.read(path)` → flush (read-your-own-writes) → `EngineClient.read(path): Promise<string | undefined>`.
- Run lifecycle: `defineFactory<O>(fn, options?)` → ALS RunContext → fn → `dialects.drain()` → `flush()` → `commit()`/`discard()`; all-or-nothing.
- **Runner flow (stdio-engine-client)**: engine spawns `src/bin/pbuilder-runner.ts` (or bootstrap dynamic-imports `enterBridge`) → fd-1 frame writer captured + console redirected FIRST → `runRunner(argv, io)`: parse argv (RUN-01) → resolve input inline/file (RUN-04, 10 MiB cap, fail-closed JSON parse with line/column locator, no raw-content echo) → await versioned `ready` greeting (WPS-02, fail-at-greeting; transport faults at greeting classify exit 3, structural/version misses exit 1) → single-instance probe (SEC-07, before import) → `import(factoryUrl)` (RUN-07: module-load failure exit 1 vs top-level throw exit 4 — BuildMessage/ResolveMessage/AggregateError shapes handled name-based, Bun-verified) → export resolution + double-wrap check (RUN-03/06) → `defineFactory(fn, {packageDir: dirname(factory)})` (RUN-05) → `run(input, {client: new StdioEngineClient(channel)})` → exit 0, or classify (EXC-01: 1 validation, 2 emit-rejection/intent-refusal, 3 transport, 4 crash; EXC-02: never reads .cause).
- **Wire flow (per reverse callback)**: `StdioEngineClient.#call` enqueues on the internal `#chain` (single-in-flight) → mints `s<n>` id → writes ONE request frame → `#awaitResponse` races the matching read loop against the injectable timeout — timeout abandons the call without consuming the successor's frame (dangling-read adoption); non-matching frames (unknown type, stale/foreign id) are wire-silently discarded with a bounded stderr note (WPS-03). `emit` additionally pre-checks `exceedsEmitBatchBudget(batch)` LOCALLY (over-budget: EmitRejection "cap", nothing written, no id minted) and hands the exactly-once-serialized body to the writer via `withPreserializedBody`. Host error envelopes: ir.emit → reconstructed `EmitRejection` (failedIndex precondition, degrade-to-unknown); tree.read/ir.commit/ir.discard → `IntentRejectedError` (a refusal is not an absence — read trichotomy stays three-way).
- **Cap seam (two legs, one comparison)**: OUTBOUND — `EMIT_BATCH_BUDGET_BYTES = BATCH_CAP_BYTES − EMIT_FRAME_ENVELOPE_ALLOWANCE_BYTES` (allowance derived by measuring the envelope shape with the longest possible id, never a literal) via `exceedsEmitBatchBudget`, shared by StdioEngineClient + ContractFake (FEH-01) + scaffold expander/classify-transport; INBOUND — raw declared length prefix checked reject-before-alloc in FrameReader via `exceedsBatchCap(number)`. The single `> BATCH_CAP_BYTES` comparison lives in wire.ts's `exceedsBatchCap` (fit-32); `>` never `>=` (exactly-at-cap is WITHIN).
- **Dialect flow (stage-5/5b, ADR-0037 + amendment)**: `find(path)` → `createDialectHandle(ast, ops, path)` returns a thenable `Handle`; each op/`.modify(fn)`/write verb enqueues a step on `#tail` and self-registers on `ctx.dialects`; N edits coalesce to ONE `modify` (mutation gate: a true no-op emits ZERO directives); a global mid-chain read/flush produces the exactly-two-modifies split; run-scoped poison flag fails a run closed first-wins; `dialects.drain()` awaits every handle before flush. `.replaceContent()` rejects while this handle's own AST-op directive is pending undrained; `.modify(fn)` coalesces freely.
- **Conformance / run-boundary validation / codegen / error attribution / dry-run** — unchanged from prior baselines (stage-5b/bare-factory postures hold).
- **Scaffold/by-reference flow (schematic-local-files)**: `scaffold(args)` → walk (bounded, no symlink-dir descent) → containment (validate BEFORE any content read, dual-anchor realpath ceiling) → filename-pipeline (pure rename→token→`.template`-strip + glob) → classify-transport (stat-size gate → UTF-8/null-byte sniff → serialized-budget verdict → by-value `create` or by-reference `copyIn`; render-request fail-loud) → expander (serialized-size chunked flush before `EMIT_BATCH_BUDGET_BYTES`). `copyIn` always by-reference, `void`-returning.
