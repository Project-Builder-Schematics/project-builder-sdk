# Architecture — project-builder-sdk
Updated: 2026-07-14 (post stage-6-release-shape, additive refresh)

## Layers
- @pbuilder/sdk [lib • TypeScript • Bun (no app framework)] — repo root (single-layer library)
  - **src/dialects/ (first dialect leaf, stage-5; WIDENED stage-5b)**: `src/dialects/typescript/` — the FIRST real dialect, published at `@pbuilder/sdk/typescript`. `index.ts` (the `find(path)` entry verb + op-pack composition via `withOps`), `ast.ts` (the ts-morph `parse`/`print` pair — the ONLY module that imports ts-morph), `ops.ts` (**FIVE structured ops, stage-5b**: `addImport`, `addFunction`, `addVariable`, `addClass`, `removeImport`, plus a shared `assertNoCollision` predicate `addFunction`/`addVariable`/`addClass` all call). Leaf-isolated: nothing under `src/commons/**` or the shared kit reaches it; FIT-02 forbids a dialect importing a sibling dialect.
  - **src/core/dialect-handle.ts (coalescing seam, stage-5; WIDENED stage-5b)**: the kit-internal handle factory (`createDialectHandle`) that owns ADR-0006's seam — read-through-parse once, hold ONE live AST across a chain, serialize once (lazy, memoized) at flush. Thenable (private `#tail` promise queue); registers itself on the run-boundary join. **Stage-5b adds**: a shared `#invokeContained` containment wrapper for `runOp`/`runRaw`; a mutation-gated `#ensureOpen` that re-baselines against the LAST EMITTED text (skips registering a directive for a true no-op, e.g. `removeImport` on an absent binding — REQ-TSD-08.4); a module-scope `WeakMap<object, string>` path channel (`astHandlePaths` + exported `handlePathFor(ast)`) that recovers an op's own handle path keyed on the LIVE AST INSTANCE. Kit-internal, no public subpath, not fitness-pinned (only the PUBLIC `find` + the `Handle<State,Ast,Ops>` TYPE are frozen).
  - **src/core/dialect-error.ts (contained-error module, NEW stage-5b)**: kit-internal, no public symbol, out of every barrel/subpath. `dialectError(tail): Error` factory + `isContained(err): boolean`, backed by a module-private `WeakSet<Error>` brand — the ONLY containment discriminator (never a `message.startsWith(prefix)` match, which a foreign error could spoof or a legitimate message could coincidentally match).
  - **src/core/deep-equal.ts (structural equality, NEW stage-5b)**: kit-internal, no public symbol. `Object.is`-based recursive structural equality (arrays + plain objects), extracted from `src/testing/contract-fake.ts`'s prior duplicate and now the SOLE implementation consumed by both `src/conformance/index.ts` (REQ-DC-04 seam-serializability) and `src/testing/contract-fake.ts` (round-trip fidelity check).
  - **src/core/schema/ (stage-4)**: the shared schema cluster (parse/validate/locate/emit-mapping) behind the IR seam, consumed by both runtime (run-boundary check) and the bin (codegen).
  - **src/testing/ (author-testing sub-surface, stage-4b)**: the THIRD audience. `runFactoryForTest` facade + the normative `ContractFake` (relocated here) + shared rejection-message dictionary. Shipped via `./testing`, in-memory only (no fs, no real engine).
  - **src/scaffold/ (folder-scaffold + by-reference-copy leaf, NEW `schematic-local-files`)**: the FIRST disk-read + directive-emit combiner — reads real package-local files/folders and lowers them to wire directives. Six files: `index.ts` (`readTemplateFile`/`runCopyIn` orchestrators, thin `commons` entry points), `walk.ts` (folder enumeration; `lstat`-based symlink-dir non-descent, never `stat`; 10 000-entry bound, FSC-09), `filename-pipeline.ts` (pure rename→token→`.template`-strip + include/exclude glob + intra-collision detection, FSC-03/05/08), `classify-transport.ts` (the classifier: stat-size gate → whole-file UTF-8/null-byte sniff → serialized-budget verdict → by-value/by-reference; render-request fail-loud carve-out, CCL-01..06; the sniff is a function within this module, not a separate file), `containment.ts` (ceiling resolution + source/dest guards, the four `source-*` AuthoringReason throws, PRC-01..09), `expander.ts` (scaffold algorithm + serialized-size chunked flush, REQ-04/05). Reached ONLY by thin `src/commons/index.ts` wrappers via `currentContext()` — one-way leaf, no reverse dependency, guarded by **FIT-22**. `DirectiveFactory` stays pure, gaining only a `copyIn` lowering.
  - **bin/ (build-time tooling, stage-4)**: `bin/pbuilder-codegen.ts` + `bin/emit-type.ts` — a zero-dep codegen CLI shipped via `package.json#bin`, OUTSIDE `src/` (FIT-15 bin→core one-way rule).

├─◄── author's factory package        [reads <package-dir>/schema.json · fs, write-contained to project root]

◄── inbound · ──► outbound · ◄► bidirectional
Note 1: ts-morph is the SDK's FIRST runtime `dependency` (exact-pinned `28.0.0`, committed `bun.lock`). It is LEAF-ISOLATED — imported only by `src/dialects/typescript/{ast,index,ops}.ts`; FIT-01's transitive import-graph walk proves it never reaches `src/commons/**` at any depth.
Note 2: the `./typescript` dialect edge is IN-PROCESS (a library call), not the engine wire. Only final `[]byte` (the printed AST) crosses the IR seam as a `modify` directive's `content`; the live ts-morph `SourceFile` never does.
Note 3: `./conformance`'s `run-vehicle.ts` is a kit-internal, in-memory transport that satisfies the port SHAPE via a LOCAL structural interface — it never names the `EngineClient`/`EmitRejection` port type (FIT-10) and is never re-exported (FIT-08). It is a SEPARATE, deliberately thin implementation from `src/testing/contract-fake.ts` (each has a distinct purpose; neither is the other).
Note 4: `./testing` and `pbuilder-codegen` retain their stage-4/4b postures — no engine edge, in-memory/write-contained I/O only.

## Pattern
- @pbuilder/sdk: custom — layered modules behind an IR-seam port, plus dialect leaves
  Evidence: `src/commons/` (author verbs) → `src/core/` (Session, DirectiveFactory, handle-state, dialect-handle) → `src/core/wire.ts` (Directive shapes) → `src/core/engine-client.ts` (single `EngineClient` port); `src/dialects/`, `src/dry-run/`, `src/testing/`, `src/scaffold/` (NEW `schematic-local-files`) sit beside `core` as deliberately-isolated leaves. No `adapters/`/`ports/`/`usecases/`/`domain/` folders; the seam is a single interface, not a hexagonal scaffold.
- **`src/dialects/typescript/` cluster (dialect leaf, stage-5; WIDENED stage-5b — REQ-TSD-09/10/11)**: an author-facing dialect leaf. `index.ts` composes the sanctioned kit surface ONLY (`defineDialect`/`defineOpPack`/`withOps` from `core/define-dialect.ts`) — no port-internal machinery imported anywhere in the tree (FIT-08/FIT-10). `ast.ts` is the sole ts-morph importer: a FROZEN `ManipulationSettings` + a CONTENT-derived `newLineKind` (never host-OS), a fresh `Project` per parse (never a module-level singleton), no language-service formatter, and a WeakMap-tracked BOM re-prepend (ts-morph@28.0.0's `getFullText()` strips a leading BOM — the module owns byte-exact BOM round-trip). `ops.ts` now carries FIVE ops: `addImport` (merges into an existing same-module clause, else inserts fresh; idempotent), `addFunction`/`addVariable`/`addClass` (insert a new top-level declaration, `{export }` prefix from `options?.export`), and `removeImport` (destructive — drops a named binding, including from a multi-specifier import, ADR-0039). `addFunction`/`addVariable`/`addClass` share one `assertNoCollision(ast, name, opName)` predicate (ADR-0039 owner ruling #4): collides on any VALUE-namespace declaration (function/const/let/var/class/enum/namespace) or a named-import binding sharing `name` — `type`/`interface` are EXEMPT (legal TS coexistence). Syntactic search only (no language service), SourceFile-wide, not scope-aware.
- **`src/core/dialect-handle.ts` (coalescing seam, stage-5, ADR-0037; WIDENED stage-5b, ADR-0037 amendment)**: a `DialectHandleController` owns `#tail` (a private promise queue), the ONE live AST, and ONE open lazy-getter `modify` directive whose `content` resolves exactly once at flush. `#ensureOpen()` re-registers a FRESH directive by IDENTITY check against `session.pendingSnapshot()` (via `Session.isPending`, NEW stage-5b) — turning a global mid-chain flush into the exactly-two-modifies split (FIT-19). An EAGER shadow-catch is attached to `#tail` in the same turn each op chains, closing the pre-drain `unhandledRejection` window. Inherited write verbs (`modify`/`rename`/`move`/`copy`/`remove`) are RE-OWNED here (enqueued on `#tail` to preserve author order), returning the same thenable handle. This is EXECUTOR LATITUDE — only the public `find` + `Handle` TYPE are frozen.
  **Stage-5b additions** (ADR-0037 amendment): (1) a SHARED `#invokeContained(fn, foreignTail)` wraps both `runOp` and `runRaw` — awaits a thenable result INSIDE the try (an async op's rejection is contained, never unhandled) and on catch discriminates by the `isContained` WeakSet brand: an author's own `dialectError` reject is RETHROWN VERBATIM, any foreign error is wrapped fresh (no double-wrap, no `.cause`); (2) a run-scoped, first-wins POISON FLAG `RunContext.runFailure?` (fail-closed, `context.ts`) — every step and every `read()` routes through one `#chain` wrapper that checks `ctx.runFailure` at entry and sets it FIRST-WINS on catch, so a DIFFERENT handle's step is never attempted as a fresh operation once any handle in the run has failed (guarded one-way by FIT-21: `context.ts` takes NO import from `dialect-handle.ts`); (3) a MUTATION GATE in `#ensureOpen` — registration only happens if the op's print actually differs from `#lastEmittedText` (seeded at `#ensureLive`, re-baselined on drain-detection), so a true no-op emits ZERO directives (REQ-TSD-08.4); (4) a module-scope `WeakMap<object, string>` PATH CHANNEL (`astHandlePaths`, set once in `#ensureLive`, exported reader `handlePathFor(ast)`) — the shipped mechanism by which an op recovers its handle's author-facing path for its own `dialectError()` message, keyed on the LIVE AST INSTANCE rather than a runtime argument position. **This replaces a trailing-positional-arg threading design considered during design (never implemented)** — a positional path arg collided with an op's own optional trailing `options` param whenever the author omitted it, silently rendering `"on \"undefined\""` in collision messages; the WeakMap channel is arity-independent by construction (final-verify fix F-1, council-architect-approved). `runModify` (S-002, ADR-0039) rejects with a `dialectError` when this handle's own AST-op directive is still pending undrained — superseding the prior silent last-write-wins; `.read()` remains the documented escape (drains the pending directive first).
- **`src/conformance/` cluster (REAL bodies, stage-5, ADR-0012 amendment; WIDENED stage-5b, ADR-0012 amendment clause 4)**: `testDialect`/`testOpPack` graduated from throwing stubs to real `Promise<void>` bodies that drive REAL runs through `defineFactory` against the kit-internal `run-vehicle.ts` (in-memory transport). DC-01..05 assertions observe an EMITTED, coalesced batch — ADR-0012 forbids mocking/bypassing the emit seam, so a real run is mandatory. `run-vehicle.ts` is internal-use only, never re-exported. **Stage-5b (REQ-DC-06/REQ-DC-08)**: SIX mandatory adversarial samples (empty file, comment-only file, a 4 MiB serialized-boundary sample, CRLF, UTF-8 BOM, duplicate-target import) are injected ADDITIVELY into every `testDialect` AND `testOpPack` run — no fixture field can opt out (compile-level guarantee). A shared `runRoundTripProbe` helper runs the real-base-dialect probe (REQ-DC-08: a stub/mock `parse` returning nullish or the input unchanged fails BEFORE the round-trip assertion) against `fixture.dialect.ast` (testDialect) or `fixture.baseDialect.ast` (testOpPack, run LAST, after the fixture's own exercises). **Leaf-isolation rule is a DOCUMENTED LIMIT**: the conformance kit runs no static or runtime scanner proving a dialect never imports outside its own leaf — `docs/authoring-a-dialect.md` states this explicitly; the SDK's OWN shipped `./typescript` dialect is proven leaf-isolated only by this repo's own `FIT-01` transitive-import-graph walk, which a third-party dialect package does not get for free.
- **`src/core/schema/` cluster (stage-4)**: cohesive core module behind the IR seam (array model, hand-rolled JSON locator), shared by runtime + bin.
- **`src/testing/` cluster (stage-4b)**: author-facing test-harness leaf reusing the SAME IR seam; `contract-fake.ts` is the sole normative `ContractFake implements EngineClient` (FIT-18 parity-by-identity).
- **Run-boundary validation site (stage-4, ADR-0029/0030)**: `context.ts` `validateAtRunBoundary()` + `checkReservedNames()` run pre-`als.run`; does NOT split the engine boundary (invariant #5 holds at the wire).

## Interconnection
- @pbuilder/sdk → Project Builder engine: JSON-RPC over stdio (planned) via the `EngineClient` port (`src/core/engine-client.ts`). No real client ships in `src/`; the only normative implementation is the contract fake at `src/testing/contract-fake.ts`. Wire methods `ir.emit` / `tree.read` are not yet on the engine (ROADMAP §6).
- **@pbuilder/sdk/typescript → ts-morph (NEW stage-5)**: an in-process library call. `src/dialects/typescript/ast.ts` imports `Project`/`NewLineKind`/`QuoteKind`/`IndentationText`/`SourceFile` from ts-morph and constructs a fresh in-memory-FS `Project` per parse. ts-morph is a plain, EXACT-pinned `dependencies` entry (`28.0.0`, ADR-0038), committed `bun.lock`, published with npm provenance.
- **@pbuilder/sdk/conformance → run-vehicle (stage-5; WIDENED stage-5b)**: in-process. `conformance/index.ts` imports `createRunVehicle` from `./run-vehicle.ts` (internal), builds a fresh isolated in-memory transport per exercise, and drives a real `defineFactory` run through it, collecting emitted batches for content assertions. Stage-5b's read-boundary-split assertion (REQ-DC-05.2) additionally injects a mid-chain `handle.read()` after the first op in a re-run, forcing the SAME global flush-before-read (ADR-0015) a live author chain would hit.
- @pbuilder/sdk/testing (facade) → ContractFake: in-process method calls only (stage-4b). NO network, NO fs, NO engine.
- pbuilder-codegen (bin) → author's factory package: filesystem read of `schema.json`, write of `schema.generated.ts`, write-contained to the invoking project root (stage-4). NO network, NO engine, NO target-tree access.

## Build / Deploy
npm package (`@pbuilder/sdk`, public, subpath exports) built by a TWO-STEP `build`: `tsc -p tsconfig.build.json` (emits `dist/` .js + .d.ts — incl. `dist/dialects/typescript/**`, `dist/testing/**`, `dist/conformance/**`, `dist/core/**`) THEN `bun build bin/pbuilder-codegen.ts --outfile dist/bin/pbuilder-codegen.js --target node --banner "#!/usr/bin/env node"`. `package.json#bin: { "pbuilder-codegen": "dist/bin/pbuilder-codegen.js" }`. `package.json#files: ["dist"]` ships the whole `dist/` tree. **`dependencies: { "ts-morph": "28.0.0" }` — the FIRST runtime dependency (exact-pinned, no range), committed `bun.lock`.** `devDependencies`: `@types/bun`, `expect-type`, `typescript`. `engines.bun >= 1.0.0` only (no `engines.node` — Bun-native; Node is publish/end-user-install tooling for the codegen bin's `#!/usr/bin/env node` shebang). • CI: GitHub Actions — `ci.yml` (build + `bun test` + strict typecheck) on non-main/PR; `publish.yml` on push to `main` (dev prerelease `0.0.0-dev.<sha>`, npm trusted publishing via OIDC + provenance, currently `--dry-run`).
- **Publish pipeline hardened, never fired (NEW `stage-6-release-shape`, ADR-0042)**: `publish.yml`'s `id-token: write` moved from workflow-level to the `publish` job's own `permissions` block (least privilege) and gated by a W3 repo-owner guard (`if: github.repository == '<owner>/<repo>'`) — closes a fork-reaches-OIDC hole verified live before this change. Trigger surface pinned to `push: branches: [main]` ONLY (no `pull_request*`/`workflow_dispatch`). Every `uses:` in BOTH `publish.yml` and `ci.yml` is 40-hex SHA-pinned (completes the pre-existing `setup-bun`-only convention). `--dry-run` stays on `npm publish` — removing it is gated on a still-pending GitHub Environment required-reviewers gate (go-live precondition, `openspec/pending-changes.md`). `package.json#scripts.prebuild` cleans `dist/` before every build (no stale-artifact ship risk); `tsconfig.build.json#declarationMap: false` drops `.d.ts.map` source-map files from the tarball. New convenience script `link:sdk` (`bun run build && bun link`) is the documented one-command local-consumption ritual (ADR-0041), exercised at parity by both the tarball and `bun link` e2e legs.

## Conventions
- New author verb → `src/commons/index.ts`.
- New internal kit primitive → `src/core/` (re-exported from `src/core/index.ts`, the extraction-ready `@pbuilder/sdk-kit` boundary; never from the umbrella). EXCEPTION — a port-internal core type (e.g. `EmitRejection`) stays OUT of the barrel to keep FIT-10's guard meaningful.
- **New dialect → its own `src/dialects/<name>/` leaf (NEW stage-5)**, published at `@pbuilder/sdk/<name>`. A dialect composes ONLY the sanctioned kit surface (`defineDialect`/`defineOpPack`/`withOps`) — never port-internal machinery (FIT-08/FIT-10) — and MUST NOT import a sibling dialect (FIT-02, leaf rule). Exactly ONE module per dialect may import its AST library (`ast.ts`), keeping FIT-01's commons-no-AST transitive walk clean.
- **AST-library imports are confined to a single per-dialect `ast.ts` (NEW stage-5)**; ts-morph is exact-pinned in `dependencies` and reachable only from `src/dialects/typescript/**`.
- **The coalescing dialect handle lives at `src/core/dialect-handle.ts` (NEW stage-5)** — kit-internal, no public subpath, EXECUTOR LATITUDE (its factory shape is not fitness-pinned); only the public `find` + `Handle<State,Ast,Ops>` type are frozen.
- **A dialect op needing its own handle's author-facing path (NEW stage-5b)** calls the exported `handlePathFor(ast)` from `src/core/dialect-handle.ts` — the ONLY channel; never a runtime argument position (arity-independent by design, final-verify fix F-1).

## Data flow
- Author verb → `DirectiveFactory.{op}` → `Session.buffer(directive)` → (on `read()` or run-end) → `Session.flush()` → `EngineClient.emit(Batch)` → engine.
- Read path: handle `read()` → `Session.read(path)` → `Session.flush()` (read-your-own-writes) → `EngineClient.read(path): Promise<string | undefined>`.
- Run lifecycle (stage-5 amended): `defineFactory<O>(fn, options?)` builds `RunContext { session, factory, dialects: DialectRegistryImpl }` in an `AsyncLocalStorage`, runs `fn`, then **`ctx.dialects.drain()`** → `flush()` → `commit()` on success / `discard()` on throw. All-or-nothing; a drain rejection routes through the SAME catch a flush rejection already used.
- **Dialect flow (stage-5, ADR-0037; WIDENED stage-5b, ADR-0037 amendment)**: `ts.find(path)` → `createDialectHandle(ast, ops, path)` returns a thenable `Handle`. Each op/`.raw`/write verb `#enqueue`s a step on `#tail` and self-`register`s once on `ctx.dialects`. The FIRST op `#ensureLive()` (`Session.read` → parse to ONE live AST, seeding `#lastEmittedText` + the `astHandlePaths` WeakMap entry) then, for `runOp`/`runRaw`, routes the op's own invocation through `#invokeContained` (awaits a thenable result inside the try; catches, discriminates by `isContained`, rethrows an author's own `dialectError` verbatim or wraps a foreign error fresh) before calling `#ensureOpen()`. `#ensureOpen()` first re-baselines `#lastEmittedText` if the prior open directive has already drained (detected via `Session.isPending` identity check — from EITHER this handle's own `read()` or a DIFFERENT handle's global flush-before-read), then buffers ONE lazy-getter `modify` whose `content` = `print(liveAst)`, resolved once at flush — but ONLY if the print actually differs from `#lastEmittedText` (mutation gate, REQ-TSD-08.4: a true no-op op registers ZERO directives). N edits coalesce to ONE `modify` (REQ-MC-01); a global mid-chain read/flush drops the open directive from `pendingSnapshot()`, so the next op re-registers a FRESH directive → exactly-two-modifies split (REQ-MC-02, FIT-19). Every step and `read()` additionally checks the run-scoped `RunContext.runFailure` poison flag at entry (fail-closed, set FIRST-WINS on any handle's rejection) — once any handle in the run has failed, a step on a DIFFERENT handle throws immediately rather than attempting a fresh mutate/buffer that could never commit (F2, ADR-0037 amendment, REQ-DG-07.3; one-way dependency guarded by FIT-21). `runModify` (S-002, ADR-0039) rejects with a `dialectError` while this handle's own AST-op directive is still pending undrained (REQ-MC-08) — `.read()` is the documented escape. At run end `dialects.drain()` awaits every handle's `settle()` (`Promise.allSettled`, re-throws first rejection) BEFORE flush — an unawaited chain still completes and commits; an unawaited THROWING chain rejects the run contained (frozen prefix `"dialect operation failed: "`, `.cause` always absent) instead of leaking `unhandledRejection` (FIT-20). Same-path concurrent unawaited handles are UNDEFINED BEHAVIOUR (documented, not prevented — REQ-MC-07); sequential awaited same-path handles are defined (cumulative split modifies).
- **Conformance flow (stage-5, ADR-0012 amendment; WIDENED stage-5b, ADR-0012 amendment clause 4)**: `testDialect`/`testOpPack` seed a fresh `createRunVehicle(seed)` per exercise, drive a REAL `defineFactory` run of the dialect through it, and assert the collected `emitted` batches for byte-exact round-trip / single-op fidelity / coalescing content / seam-serializability / read-boundary-split. Stage-5b additively injects SIX mandatory adversarial samples (empty file, comment-only, 4 MiB boundary, CRLF, UTF-8 BOM, duplicate-target import) into both functions' round-trip assertions, plus a real-base-dialect probe (REQ-DC-08: a stub/mock `parse` returning nullish or an unchanged string fails loud, BEFORE the round-trip check) — no fixture field can disable this (REQ-DC-06.2, `test/conformance/planted/opt-out-attempt.test.ts` proves it at the type level). `testOpPack`'s probe runs against `fixture.baseDialect.ast`, LAST, after the fixture's own per-exercise assertions.
- Run-boundary validation (stage-4), Codegen flow (stage-4), Error attribution (stage-2), Dry-run (stage-3) — unchanged from prior baselines.
- **Scaffold/by-reference flow (NEW `schematic-local-files`)**: `scaffold(args)` → `src/scaffold/walk.ts` enumerates `from` (bounded, no symlink-dir descent) → `src/scaffold/containment.ts` `validateSourceContainment` runs BEFORE any content read (PRC-08) — lexical `../`/absolute screen → segment-aware ceiling check against `packageAnchors.packageRoot` → realpath (ENOENT resolved only after the nearest EXISTING ancestor's realpath is proven in-ceiling, closing the ancestor-symlink oracle) → `lstat` allow-list (regular files only) → `src/scaffold/filename-pipeline.ts` applies rename→token→`.template`-strip + include/exclude filtering (pure, no I/O) → `src/scaffold/classify-transport.ts` classifies each surviving file: stat-size gate → whole-file UTF-8/null-byte sniff → serialized-directive-size budget (measured against the PROSPECTIVE directive, not the raw content string) → by-value (`create`) or by-reference (`copyIn`); a `.template`/`templateFile` render request failing any gate throws `invalid-input` fail-loud, NEVER silently degrades to by-reference → `src/scaffold/expander.ts` maintains a serialized-size accumulator and calls `session.flush()` before the running group would exceed `BATCH_CAP_BYTES`, starting a fresh group (REQ-04/05); run-level all-or-nothing needs no new mechanism (`commit()`/`discard()` unchanged). `copyIn(from, to, opts?)` always by-reference, never classified, `void`-returning. Destination lexical guard (PRC-09) applies to the FINAL SDK-computed destination (post-rename, post-token-translation), catching a `rename` value smuggling `../`.
